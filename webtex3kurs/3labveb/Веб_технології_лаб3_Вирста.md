## **Теоретичні відомості**

1. **Що таке Node.js**Node.js — відкрите середовище виконання JavaScript поза браузером, побудоване на рушії V8 (Google Chrome). Дозволяє використовувати JavaScript для серверної частини та скриптів автоматизації.

2. **Особливості Node.js**- Подієво-орієнтована архітектура (event loop).- Однопоточність у поєднанні з неблокуючими операціями.- Асинхронна модель виконання (callbacks / Promises / async‑await).- Кросплатформність (Windows / macOS / Linux).

3. **Де використовується Node.js**Серверні застосунки, REST API, реальний час (чати), мікросервіси, інструменти зборки (webpack, vite), CLI‑утиліти.

4. **npm (Node Package Manager)**Менеджер пакетів, що встановлюється разом з Node.js. Дає змогу ініціалізувати проєкт (`npm init`), встановлювати залежності (`npm install <package>`), запускати скрипти (`npm start`) тощо.

5. **Робота з модулями**- Вбудовані модулі: `fs`, `path`, `http` (`const fs = require('fs');`).- Зовнішні пакети: `axios`, `express` (`const axios = require('axios');`).

6. **Файлова система (fs)**- `fs.readFile(path, encoding, cb)` — асинхронне читання.- `fs.writeFile(path, data, cb)` — асинхронний запис.- `fs.promises` — Promise‑версії функцій (підтримують async/await).

7. **Асинхронність у Node.js**Node.js не блокує потік при вводу/виводі; результат операції приходить через callback або завершується Promise. Приклади з `async/await` значно спрощують читабельність коду.

## **Хід роботи (практична частина)**

1. **Встановлення Node.js**Завантажив та встановив Node.js з офіційного сайту: `https://nodejs.org`.Перевірка у терміналі: `node -v`, `npm -v`.

2. **Створення проєкту та файлів**Структура проєкту, що була створена для лабораторної роботи:
```
/lab3-node
├── mydata.txt
├── app.js
└── result.txt (створюється скриптом)
```

3. **Вміст `mydata.txt` (вхідні дані)**```
1. Купити хліб
2. Зробити домашнє завдання
3. Подивитись фільм
```

4. **Реалізація скрипта `app.js`**Написав Node.js скрипт, який читає `mydata.txt`, додає рядок-підпис із номером варіанту та прізвищем/іменем студента, і зберігає результат у `result.txt`.

```javascript
// app.js
const fs = require('fs');

fs.readFile('mydata.txt', 'utf8', (err, data) => {
  if (err) {
    console.error("Помилка при читанні файлу:", err);
    process.exit(1);
  }

  const updatedData = data + "\nНепарний варіант — додано студентом Вирста Андрій";

  fs.writeFile('result.txt', updatedData, (err) => {
    if (err) {
      console.error("Помилка при записі файлу:", err);
      process.exit(1);
    }
    console.log("✅ Файл result.txt створено успішно!");
  });
});
```

5. **Запуск скрипту**У каталозі проєкту виконано у терміналі команду:
```
node app.js
```
В результаті з'явився файл `result.txt` із доповненим вмістом.

6. **Вміст `result.txt` (вихідні дані після виконання скрипту)**```
1. Купити хліб
2. Зробити домашнє завдання
3. Подивитись фільм

Непарний варіант — додано студентом Вирста Андрій
```

## **Контрольні питання та відповіді**

**1. Що таке Node.js і для чого він використовується?**Node.js — середовище виконання JavaScript поза браузером. Використовується для створення серверних застосунків, CLI‑утиліт, скриптів автоматизації та інструментів зборки.

**2. Що таке npm і для чого він потрібен?**npm — менеджер пакетів для Node.js. Допомагає встановлювати та оновлювати зовнішні бібліотеки, керувати залежностями і запускати скрипти проєкту.

**3. Як перевірити версію встановленого Node.js?**У терміналі виконати команду `node -v` (або `node --version`).

**4. Чим відрізняється виконання JS у браузері та у Node.js?**У браузері JS працює у середовищі браузера з доступом до DOM, window, document. У Node.js — немає DOM, є доступ до файлової системи, мережі, процесу; модульна система інша (`require`/ESM).

**5. Які переваги має асинхронна модель роботи Node.js?**Ефективне використання ресурсів при обробці великої кількості одночасних з'єднань; неблокуючі операції I/O дозволяють обслуговувати більше запитів без створення додаткових потоків.

**6. Як у Node.js підключаються модулі?**Через `require` (CommonJS) або `import` (ES Modules). Приклад: `const fs = require('fs');`

**7. Які функції модуля fs ви знаєте для роботи з файлами?**- `fs.readFile`, `fs.readFileSync` — читання файлів.- `fs.writeFile`, `fs.writeFileSync` — запис файлів.- `fs.appendFile` — додавання в кінець файлу.- `fs.readdir` — читання списку файлів у директорії.- `fs.promises` — Promise‑версії функцій.
